# 식단관리 (Diet Manager)

개인용 식단 관리 **PWA**입니다. 브라우저에서 앱처럼 설치해 쓰고, 데이터는 서버 없이 **브라우저(localStorage)** 에만 저장됩니다.

## 주요 기능

- **식사 기록** — 메뉴 이름 입력 → AI가 칼로리/탄단지/나트륨 추정, 음식 사진 → AI 이미지 인식, 직접 수정 가능
- **나만의 메뉴** — 자주 먹는 18개 메뉴 기본 내장, 탭 한 번으로 바로 기록, 커스텀 메뉴 추가
- **오늘 대시보드** — 칼로리 원형 게이지, 탄단지 도넛 차트, 나트륨 막대(초과 경고), 남은 칼로리 안내, 미기록 리마인더
- **통계** — 주간 칼로리 추이, 체중 기록·변화 그래프, FMD 사이클(25일 일반식 + 5일 FMD) 30일 캘린더
- **알림** — 앱이 열려 있는 동안 식사 시간·나트륨 초과 브라우저 알림
- **설정** — 목표 칼로리/나트륨, 식사 시간, FMD 시작일, AI 모델 선택, 데이터 백업(JSON)/초기화

## 기술 스택

- Vite + React 19 + TypeScript
- Tailwind CSS v4
- recharts (차트)
- vite-plugin-pwa (설치형 PWA / 서비스 워커)
- 저장소: 브라우저 localStorage (서버·계정 없음)
- AI 추정: `@anthropic-ai/sdk` — 브라우저에서 사용자 본인 키로 Claude API 직접 호출

## 개발

```bash
bun install      # 또는 npm install
bun run dev      # 개발 서버 (http://localhost:5173)
bun run build    # 프로덕션 빌드 (dist/)
bun run preview  # 빌드 결과 미리보기
```

## AI 추정 사용법

메뉴 이름·사진 인식 기능은 **Anthropic API 키**가 필요합니다.

1. [Anthropic Console](https://console.anthropic.com/)에서 API 키(`sk-ant-...`)를 발급받습니다.
2. 앱 → **설정 → AI 추정** 에 키를 입력합니다. (키는 서버로 전송되지 않고 이 기기의 브라우저에만 저장됩니다.)
3. 모델을 선택합니다 — Opus 4.8(가장 정확) / Sonnet 5(균형) / Haiku 4.5(가장 저렴). 본인 키로 과금되므로 비용에 맞춰 고르세요.

> AI 키가 없어도 나만의 메뉴 탭 기록과 직접 입력은 그대로 사용할 수 있습니다.

## 배포

정적 사이트라 어디에나 올릴 수 있습니다. `bun run build` 후 `dist/` 를 Vercel·Netlify·GitHub Pages 등에 배포하세요. 아이폰 Safari에서 **홈 화면에 추가**하면 앱처럼 실행됩니다.

## 알림 관련 한계

브라우저 알림은 **앱(탭)이 열려 있을 때만** 동작합니다. 앱을 완전히 닫은 상태에서 정해진 시각에 울리는 백그라운드 푸시는 별도 서버(푸시 서비스)가 필요해 이 버전에는 포함되지 않습니다. iOS Safari는 백그라운드 푸시 제약이 있습니다.

## 데이터

모든 기록은 이 브라우저에만 저장됩니다. 기기를 바꾸거나 브라우저 데이터를 지우면 사라지므로, **설정 → 데이터 → 백업 내보내기**로 주기적으로 JSON 백업을 받아두세요.
